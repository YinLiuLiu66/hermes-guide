首次配置

  1. 运行设置向导
    hermes setup

    交互式引导你配置模型、终端、工具等。也可以分步执行：

    hermes setup model # 配置模型和提供商 hermes setup terminal # 配置终端后端 hermes setup tools # 配置工具集
  2. 选择模型
    hermes model # 交互式模型/提供商选择器

    支持 OpenRouter、Anthropic、Google Gemini、DeepSeek 等。需要对应提供商的 API Key。

  3. 添加 API 密钥

    编辑 ~/.hermes/.env 文件，添加你的 API Key：

    # 至少配置一个提供商的密钥 OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxx # 或者 ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx # 或者 GOOGLE_API_KEY=AIzaxxxxxxxxxx
  4. 健康检查
    hermes doctor # 检查配置和依赖 hermes doctor --fix # 自动修复可修复的问题

开始使用

hermes # 启动交互式对话 hermes chat -q "你好" # 单次提问
提示：中国用户如果 GitHub 访问受限，安装时可能需要配置代理或使用镜像。安装脚本支持通过环境变量设置代理：https_proxy=http://127.0.0.1:7890 bash install.sh

基础使用

交互式对话

直接运行 hermes 即可进入交互式对话。Hermes 可以使用各种工具：执行命令、读写文件、搜索网页、分析图片等。

工作目录

Hermes 默认在当前目录工作。你可以指定工作目录：

cd ~/my-project hermes # 在 my-project 目录下工作

这样 Hermes 的文件操作、Git 操作等都会在此目录下进行。

配置 Telegram

1. 创建 Bot
在 Telegram 中找 @BotFather，发送 /newbot，获取 Bot Token。
2. 配置 Token
# 编辑 ~/.hermes/.env TELEGRAM_BOT_TOKEN=123456789:ABCdefGhIjKlMnOpQrStUvWxYz
3. 启动网关
hermes gateway setup # 交互式选择平台 hermes gateway install # 安装为后台服务 hermes gateway start # 启动服务

配置飞书

1. 创建飞书应用
在飞书开放平台创建应用，获取 App ID、App Secret 和 Verification Token。
2. 配置环境变量
# 编辑 ~/.hermes/.env FEISHU_APP_ID=cli_xxxxxxxxxx FEISHU_APP_SECRET=xxxxxxxxxx FEISHU_VERIFICATION_TOKEN=xxxxxxxxxx
3. 启动
hermes gateway setup # 选择飞书 hermes gateway install hermes gateway start
飞书需要配置事件回调 URL，详见官方文档。

配置 Discord

1. 创建 Discord 应用
在Discord Developer Portal创建应用，添加 Bot，获取 Token。
2. 重要：开启 Intent
在 Bot 设置中开启 Message Content Intent，否则 Bot 无法读取消息内容。
3. 配置并启动
# ~/.hermes/.env DISCORD_BOT_TOKEN=MTxxxxxxxxxx.xxxxxx.xxxxxxxxxxxxx hermes gateway setup # 选择 Discord hermes gateway install hermes gateway start

管理网关服务

hermes gateway status # 查看状态 hermes gateway start # 启动后台服务 hermes gateway stop # 停止服务 hermes gateway restart # 重启服务 hermes gateway run # 前台运行（调试用） hermes gateway list # 列出所有 Profile 的网关状态
SSH 环境注意：SSH 断开后后台服务可能停止。执行 sudo loginctl enable-linger $USER 保持服务运行。WSL2 需要在 /etc/wsl.conf 中设置 systemd=true。

Skills 系统

什么是 Skill？

Skill 是 Hermes 的「程序性记忆」—— 可复用的操作流程文档。当 Hermes 解决了一个复杂问题、发现了有效的工作流程、或被你纠正了做法，它可以将这些经验保存为 Skill，在未来的会话中自动加载使用。

Skills 随时间积累，让 Hermes 在你的特定任务和环境中变得越来越好。

Skill 来源

来源说明
内置 Skill随 Hermes 安装的官方技能
Hub 安装从 Skill 仓库安装的社区技能
Agent 创建Hermes 在工作过程中自动创建的技能
自定义用户手动编写的 SKILL.md 文件

Curator — 自动维护

Hermes 内置 Curator 系统，自动管理 Agent 创建的 Skills 生命周期：

  • 追踪每个 Skill 的使用频率和活动时间
  • 标记长期未使用的 Skill 为「过时」
  • 自动归档（从不删除），保留备份
  • 固定（pin）的 Skill 不受自动管理影响
hermes curator status # 查看 Curator 状态 hermes curator run # 手动执行维护 hermes curator pin <name> # 固定技能（不被归档）

创建自定义 Skill

Skill 是一个 Markdown 文件（SKILL.md），放在 ~/.hermes/skills/ 目录下。包含 YAML 前置信息和 Markdown 正文：

--- name: my-skill description: "技能描述，说明何时使用" tags: [tag1, tag2] --- # 技能标题 ## 触发条件 描述何时应该加载这个技能。 ## 操作步骤 1. 第一步... 2. 第二步... ## 注意事项 常见坑和解决方案。

基本设置流程

  1. 运行交互式设置
    hermes gateway setup

    选择要配置的平台，按提示输入 Token / 密钥。

  2. 配置环境变量

    将 API Token 和密钥添加到 ~/.hermes/.env。具体变量名取决于平台（参见上表各平台的说明）。

  3. 安装并启动
    hermes gateway install # 安装为系统服务 hermes gateway start # 启动服务 hermes gateway status # 检查状态

管理命令

hermes gateway status # 查看状态 hermes gateway start # 启动 hermes gateway stop # 停止 hermes gateway restart # 重启 hermes gateway run # 前台运行（调试用） hermes gateway list # 列出所有 Profile 的网关 hermes gateway uninstall # 卸载系统服务

进阶功能

Delegation — 子任务委派

Hermes 可以将任务委派给子智能体，子智能体在隔离的上下文中运行，完成后返回结果摘要。

# 配置委派参数 hermes config set delegation.model anthropic/claude-sonnet-4 hermes config set delegation.max_iterations 50
  • leaf 角色（默认）：专注执行，不能再委派
  • orchestrator 角色：可以创建自己的子工作者
  • 通过 delegation.max_concurrent_children 控制并发数

常见问题

工具不可用

# 检查工具是否启用 hermes tools list # 启用需要的工具 hermes tools enable <toolset> # 开始新会话使变更生效 /reset

某些工具需要环境变量（如浏览器工具需要 Browserbase API Key），检查 ~/.hermes/.env。

模型/提供商问题

hermes doctor # 全面检查 hermes auth list # 检查凭证 hermes auth add <provider> # 重新认证
  • 确认 ~/.hermes/.env 中的 API Key 正确
  • Copilot 403 错误：gh auth login 的 Token 不适用于 Copilot API，需要通过 hermes model → GitHub Copilot 走 OAuth 流程

配置变更不生效

  • 工具/技能变更：在对话中输入 /reset 开始新会话
  • 配置文件变更：CLI 中退出重新运行，Gateway 中使用 /restart
  • 代码变更：重启 CLI 或 Gateway 进程

Skill 不显示

hermes skills list # 确认已安装 hermes skills config # 检查平台启用状态 # 手动加载 /skill <name>

Gateway 问题

# 查看日志 tail -20 ~/.hermes/logs/gateway.log
问题解决
SSH 断开后 Gateway 停止sudo loginctl enable-linger $USER
WSL2 中 Gateway 不稳定在 /etc/wsl.conf 设置 systemd=true
Gateway 崩溃循环systemctl --user reset-failed hermes-gateway
Discord Bot 无响应开启 Message Content Intent
Slack Bot 仅私聊可用订阅 message.channels 事件

中国网络环境

  • 安装时配置代理：https_proxy=http://127.0.0.1:7890 bash install.sh
  • Git clone 失败：使用 GitHub 镜像或配置 git config --global url."https://ghproxy.com/https://github.com/".insteadOf "https://github.com/"
  • 推荐使用硅基流动（SiliconFlow）等国内提供商作为 OpenRouter 替代
  • 配置 model.context_length 可跳过 OpenRouter 的元数据请求（网络受限时有用）

Windows 专属问题

  • 换行：使用 Ctrl+Enter（而非 Alt+Enter，后者会全屏）
  • HTTP 400 错误：config.yaml 可能保存为 UTF-8 BOM 格式，用 hermes config edit 重新保存
  • 路径：Hermes 接受正斜杠 /，建议统一使用
